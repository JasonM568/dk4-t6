## P.2｜模組 A：Instruction 解剖練習

### 五層架構速查

| 層次 | 名稱 | 核心問題 |
|------|------|---------|
| 第一層 | 角色定義 (Persona) | 你是誰？語氣是什麼？ |
| 第二層 | 任務範圍 (Scope) | 你做什麼？不做什麼？ |
| 第三層 | 行為規則 (Rules) | 有哪些固定要做或絕對不做的事？ |
| 第四層 | 輸出格式 (Output) | 答案長什麼樣子？ |
| 第五層 | 情境處理 (Edge Cases) | 遇到特殊情況怎麼辦？ |

---

### 練習：標出每句話的層次

閱讀以下 Instruction，在每一句話旁邊的括號填入層次代號（Persona / Scope / Rules / Output / Edge）

```
你是一位專精於台灣勞動法規的諮詢助理，        （         ）
服務對象是一般勞工，非法律專業人士。           （         ）

你的服務範圍：                                 （         ）
- 解釋勞動基準法相關條文
- 說明勞工申訴管道與流程
- 提供一般性法律資訊與參考方向

你不處理以下事項：                             （         ）
- 具體訴訟策略或法庭建議
- 雇主方的問題
- 非勞動法範疇的法律問題

回答語言：繁體中文                             （         ）
回答長度：200-400 字，用條列式呈現重點          （         ）
如無法協助，回應：
「建議您諮詢執業律師或向勞工局申訴。」          （         ）
每次回答結尾加上：
「本資訊僅供參考，不構成法律意見。」            （         ）
```

---

### 思考題（討論用）

1. 如果拿掉「你不處理以下事項」這段，GPT 會怎樣？

   你的答案：_______________________________________________

2. 如果沒有字數限制，使用體驗會有什麼影響？

   你的答案：_______________________________________________

3. 你認為哪一句話最關鍵？為什麼？

   你的答案：_______________________________________________

## P.3｜模組 B 模板：知識顧問型 GPT

**適合場景：** 專業問題 Q&A（法律、醫療、財務、技術支援）  
**核心重點：** 邊界要清楚，免責聲明要有

### 填寫你的 Instruction

```
你是一位專精於 [你的專業領域] 的 ________________。
服務對象是 ________________。

你的服務範圍：
- ________________
- ________________
- ________________

你不處理以下事項：
- ________________
- ________________
- ________________

回答語言：________________
回答長度：________________
如無法協助，回應：「________________」
每次回答 [開頭/結尾] 加上：「________________」
```

### 注意事項提醒
- 「不處理」的部分至少寫 2 條，越具體越好
- 免責聲明每次都要出現，不要說「視情況加上」

---

## P.4｜模組 B 模板：流程執行型 GPT

**適合場景：** 有固定步驟的任務（履歷優化、文件整理、流程引導）  
**核心重點：** 順序要固定，輸出格式要具體

### 填寫你的 Instruction

```
你是一位專業的 [角色名稱] ________________。

工作流程（每次嚴格依照這個順序）：

第一步：收到使用者的 [輸入內容] 後，先詢問：
  - ________________
  - ________________

第二步：[分析/整理/處理]，並指出 [數量] 個重點。
  格式：
  【重點 1】________________
  【重點 2】________________

第三步：輸出 [最終結果]。
  每個項目必須包含：
  - ________________
  - ________________

如果使用者沒有提供足夠資訊，先問清楚再動手。
不要跳過任何步驟。
```

### 注意事項提醒
- 「先詢問再動手」這句話很重要，不要漏掉
- 輸出格式越具體（如：必須包含哪些欄位），結果越一致

## P.5｜模組 B 模板：創意產出型 GPT

**適合場景：** 文案、品牌聲音、社群貼文、創意寫作  
**核心重點：** 風格描述要精確，附上範例效果最好

### 填寫你的 Instruction

```
你是「[品牌/個人名稱]」的 [文案師/社群小編/寫手]。

品牌個性：
- ________________（用形容詞描述，如：溫暖、直接、有點幽默）
- ________________
- ________________

語氣規則：
- 不用這些詞：________________（如：超棒、必買、限時）
- 句子長度：________________（如：最多 20 字一句）
- 多用「________________」，少用「________________」

每篇 [貼文/文案] 結構：
1. ________________（如：開場情境，1-2 句）
2. ________________（如：品牌連結，1 句）
3. ________________（如：結尾邀請，1 句）

字數：________________
固定標籤：________________

範例風格（重要！貼上一段你認為最接近理想風格的文字）：
「________________
________________
________________」
```

### 注意事項提醒
- 「範例風格」是這個類型最重要的設定，不要跳過
- 風格描述越具體越好（不要只說「友善」，要說「像朋友講話，不像廣告詞」）

## P.7｜模組 D：症狀診斷對照表

當你的 GPT 出現問題，先對照這張表找原因。

| 症狀 | 最可能的原因 | 修改哪個層 |
|------|-------------|-----------|
| GPT 什麼問題都接受，沒有邊界 | Scope 沒寫「不做什麼」 | 任務範圍層 |
| 輸出格式每次都不一樣 | Output Format 沒有具體規定 | 輸出格式層 |
| GPT 語氣奇怪，不像預期 | Persona 描述太抽象 | 角色定義層 |
| 遇到模糊問題 GPT 就自己猜 | Rules 沒有「先追問再回答」規則 | 行為規則層 |
| 遇到超出範圍的問題 GPT 亂回 | Edge Cases 沒有設定 | 情境處理層 |
| 使用者不知道怎麼開始使用 | Conversation Starter 沒設計 | 非 Instruction 問題 |
| GPT 被使用者說服改變身份 | Persona 沒有加入「維持角色」規則 | 行為規則層 |
| 創意型 GPT 輸出風格不穩定 | 缺少範例（Few-shot）| 角色定義層 |

---

### 修改的黃金規則

> **一次只改一個地方**  
> 改完之後先測試，確認有效，再改下一個問題。  
> 一次改五個地方，你永遠不知道是哪個改動有效。

---

### 版本記錄習慣（建議）

每次修改前，把當前版本複製備份：

```
格式：日期_版本號_這次修改什麼
範例：20240315_v2_加入範圍限制
      20240318_v3_修改輸出格式為條列式
      20240320_v4_加入無法回答時的引導語
```

## 附錄｜Instruction 寫作速查清單

在你提交 Instruction 之前，對照這個清單：

**角色定義**
- [ ] 有說明 GPT 是誰（職稱、專業領域）
- [ ] 有說明語氣風格（正式/輕鬆/專業）
- [ ] 有說明服務對象是誰

**任務範圍**
- [ ] 有列出「做什麼」
- [ ] 有列出「不做什麼」（至少 2 條）

**行為規則**
- [ ] 有說明遇到不完整問題時如何處理
- [ ] 有說明維持角色的規則（可選）

**輸出格式**
- [ ] 有指定輸出語言
- [ ] 有指定字數或長度範圍
- [ ] 有指定格式（條列式/段落/表格）

**情境處理**
- [ ] 有說明無法回答時的固定回應語
- [ ] 有說明免責聲明（如適用）

**測試**
- [ ] 五種壓力測試都跑過一遍
- [ ] 測試記錄表已填寫完畢
